# Incident Handler's Journal
## Overview
This project was completed as part of the Google Cybersecurity Professional Certificate. The journal documents cybersecurity incidents, investigations, tools, and response concepts encountered throughout incident detection and response activities. The journal acts as a record of incident analysis and reflects how security professionals document incidents and key findings during investigations.

## Disclaimer
The scenarios documented in this journal are fictional and were created for educational purposes as part of the Google Cybersecurity Professional Certificate.

## Project Objectives
The objectives of this journal were to:
- Practice professional incident documentation
- Record incident details using structured reporting methods
- Apply the 5 W's framework during incident analysis
- Develop incident response documentation skills
- Keep a record of cybersecurity investigations and lessons learned

## Documentation Method
Each journal entry records key incident details using a structured format that includes:

- Date and entry number
- Incident description
- Tools used
- The 5 W's of incident analysis
- Additional notes and observations

This approach supports organised incident tracking and promotes consistent documentation practices.

## Skills Demonstrated
- Incident documentation

- Investigation note-taking
- Threat and attack identification
- Structured reporting
