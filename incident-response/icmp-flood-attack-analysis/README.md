# ICMP Flood Attack Analysis

## Overview
This project was completed as part of the Google Cybersecurity Professional Certificate. The activity focused on analysing a denial of service (DoS) attack using the NIST Cybersecurity Framework (NIST CSF).

The project includes:
- Incident analysis
- Risk identification
- Security improvement planning
- Detection and response recommendations
- Recovery planning

## Disclaimer
This scenario is fictional and was created for educational purposes as part of the Google Cybersecurity Professional Certificate.

## Scenario Summary
A multimedia company experienced a denial of service (DoS) attack caused by a flood of incoming ICMP packets. The attack overwhelmed the organisation’s internal network and disrupted network services for around two hours. The investigation determined that the attack entered the network through an improperly configured firewall. Following the incident, the cybersecurity team implemented additional firewall protections, IDS/IPS systems, and network monitoring tools to strengthen network security.

## Objectives
The objectives of this project were to:
- Analyse the cause and impact of the security incident
- Apply the NIST Cybersecurity Framework (NIST CSF)
- Identify weaknesses in network security controls
- Develop protection, detection, response, and recovery strategies
- Improve preparation for future incidents
