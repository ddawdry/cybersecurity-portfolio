# File Permissions Management in Linux

## Overview
This project was completed as part of the Google Cybersecurity Professional Certificate. The activity focused on reviewing and modifying file and directory permissions in a Linux environment to ensure proper authorisation and improve system security. The project shows how Linux permissions can be used to control access to files and directories using command-line tools.

## Disclaimer
This scenario is fictional and was created for educational purposes as part of the Google Cybersecurity Professional Certificate.

## Project Objectives
The objectives of this project were to:
- Review existing Linux file permissions
- Identify unauthorised access permissions
- Modify file and directory permissions using `chmod`
- Understand hidden files and directory permissions
- Apply least privilege access control principles

## Linux Commands Used
Commands used during this project include:

```bash
ls -la
chmod
```

## Skills Demonstrated
- Linux command-line usage
- File permission management
- Access control
- Authorisation management
- Hidden file handling
- Linux security fundamentals
- Least privilege principles

## Key Concepts Covered
### Viewing Permissions
Used `ls -la` to display:
- File permissions
- Hidden files
- Ownership details
- Directory permissions

### Understanding Permission Strings
Learned how to interpret Linux permission strings such as:

```text
-rw-rw-r--
```

including:
- user permissions
- group permissions
- other permissions
- read, write, and execute access

### Modifying Permissions
Used `chmod` to:
- remove unauthorised write access
- secure hidden files
- restrict directory access
