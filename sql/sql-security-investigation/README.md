# SQL Security Investigation Using Filters

## Overview
This project was completed as part of the Google Cybersecurity Professional Certificate. The activity focused on using SQL filtering techniques to investigate login activity and employee device information in an organisational database. The project demonstrates how SQL queries can support cybersecurity investigations by retrieving targeted information from large datasets.

## Disclaimer
This scenario is fictional and was created for educational purposes as part of the Google Cybersecurity Professional Certificate.

## Project Objectives
The objectives of this project were to:
- Investigate suspicious login activity
- Filter security data using SQL queries
- Analyse login attempts based on time, date, and location
- Identify employee systems needing security updates
- Practice using SQL to support cybersecurity investigations

## SQL Concepts Used
This project used multiple SQL filtering techniques, including:

```sql
AND
OR
NOT
LIKE
%
```

These operators were used to narrow search results and retrieve relevant security information from organisational databases.

## Skills Demonstrated
- SQL querying
- Data filtering and analysis
- Security investigation workflows
- Login activity analysis
- Database investigation
- Pattern matching using LIKE
- Logical filtering using AND, OR, and NOT
- Security focused data analysis

## Investigation Areas
The investigation focused on:

### Login Activity Analysis
Reviewed:
- After-hours failed login attempts
- Login attempts on suspicious dates
- Login attempts outside approved geographic locations

### Employee Device Management
Identified:
- Marketing employees in East offices
- Employees in Finance and Sales departments
- Employees outside the IT department needing updates
